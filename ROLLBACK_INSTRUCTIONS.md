# Инструкции по откату версии

## Текущая версия: v1.2.0

Эта версия включает:
- Настраиваемый отступ лимитных ордеров (`LIMIT_ORDER_OFFSET`)
- Автоматический стоп-лосс (`STOP_LOSS_OFFSET`)

## Откат на предыдущую версию (v1.1.0)

### Быстрый откат (временный)

Просмотреть код предыдущей версии без изменения текущей ветки:

```bash
git checkout v1.1.0
```

**Важно:** Это переведет вас в "detached HEAD" состояние. Для возврата к v1.2.0:

```bash
git checkout main
```

### Полный откат (постоянный)

Если нужно откатить проект на предыдущую версию навсегда:

```bash
# Создать ветку отката
git checkout -b rollback-to-v1.1.0 v1.1.0

# Или откатить main (осторожно!)
git reset --hard v1.1.0
git push origin main --force
```

⚠️ **Внимание:** `--force` перезапишет историю на сервере!

### Откат только конфигурации

Если хотите откатить только файлы конфигурации:

```bash
# Откатить example.env
git checkout v1.1.0 -- example.env env.example

# Удалить новые переменные из .env
# Вручную удалите LIMIT_ORDER_OFFSET и STOP_LOSS_OFFSET
```

## Доступные версии

- `v1.2.0` - Текущая (настраиваемый лимит + стоп-лосс)
- `v1.1.0` - Стабильная (volumeUSDT support)
- `v1.0.0-backup` - Базовая версия

## Просмотр изменений между версиями

```bash
# Сравнить v1.1.0 и v1.2.0
git diff v1.1.0..v1.2.0

# Посмотреть список измененных файлов
git diff --name-only v1.1.0..v1.2.0

# Посмотреть коммиты между версиями
git log v1.1.0..v1.2.0 --oneline
```

## Восстановление после отката

Если вы откатились и хотите вернуться:

```bash
git checkout main
git pull origin main
```

## Помощь

Если возникли проблемы:

1. Проверьте текущую версию: `git describe --tags`
2. Проверьте статус: `git status`
3. Посмотрите доступные теги: `git tag -l`
4. Свяжитесь с разработчиком для поддержки
